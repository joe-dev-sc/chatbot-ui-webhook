Overview

Installation & Setup

CDN Integration

<!-- Include CSS -->
<link href="https://your-domain.com/chatbot-widget.css" rel="stylesheet" />

<!-- Include JavaScript Module -->
<script type="module">
    import ChatbotWidget from 'https://your-domain.com/chatbot-widget.js';
    
    const chatbot = new ChatbotWidget({
        webhookUrl: 'https://your-n8n-instance.com/webhook/your-webhook-id',
        // ... other configuration options
    });
</script>

Local Installation

<!-- Include local files -->
<link href="./chatbot-widget.css" rel="stylesheet" />
<script type="module" src="./chatbot-widget.js"></script>

<script type="module">
    import ChatbotWidget from './chatbot-widget.js';
    
    const chatbot = new ChatbotWidget({
        webhookUrl: 'https://your-webhook-url.com',
        title: 'Customer Support'
    });
</script>

Auto-Initialization with Data Attributes

You can automatically initialize the widget using HTML data attributes:

<div 
    id="my-chatbot" 
    data-chatbot-auto-init="true"
    data-webhook-url="https://your-webhook-url.com"
    data-title="Customer Support"
    data-primary-color="#e67e22"
    data-position="bottom-right"
    data-welcome-message="Welcome! How can we help you today?"
    data-title-logo="https://example.com/logo.png"
    data-tooltip-text="Need help? Click here to chat!"
></div>
Note: Make sure your n8n webhook is configured to accept POST requests and returns a JSON response with either an 'output' or 'message' field. The widget now sends a payload with both 'message' and 'chatId' for session tracking.

Configuration Options

The chatbot widget accepts a comprehensive set of configuration options to customize its appearance and behavior.

Option Type Default Description
webhookUrl Required string '' The n8n webhook URL where messages will be sent
containerId Optional string 'chatbot-widget' ID of the container element where the widget will be rendered
title Optional string 'Chat with us!' Title displayed in the chat header
placeholder Optional string 'Type your message...' Placeholder text for the input field
primaryColor Optional string '#007bff' Primary color for the widget theme (hex format)
chatBackgroundColor Optional string '#f8f9fa' Background color for the chat area
headerTextColor Optional string '#ffffff' Text color for the header
position Optional string 'bottom-right' Widget position: 'bottom-right', 'bottom-left', or 'inline'
welcomeMessage Optional string 'Hello! How can I help you today?' Initial welcome message displayed to users
errorMessage Optional string 'Sorry, something went wrong. Please try again.' Error message shown when requests fail
footerText Optional string 'Powered By Bot' Text displayed in the widget footer
botIcon Optional string null URL to custom bot icon (supports images, GIFs, etc.)
titleLogo Optional string null URL to logo displayed in the chat header next to the title
showLogoInChat Optional boolean true Whether to show the title logo next to bot messages in the chat
tooltipText Optional string 'Here is your chatbot' Tooltip text displayed when hovering over the toggle button
maxRetries Optional number 3 Maximum number of retry attempts for failed requests
retryDelay Optional number 2000 Delay between retry attempts (milliseconds)
suggestedQuestions Optional array [] Array of suggested questions to display to users
maxInputLength Optional number 500 Maximum character length for user input
maxInputWarning Optional string 'Message is too long. Please keep it under {max} characters.' Warning message for input length validation (use {max} placeholder)
storageKey Optional string 'chatbot-history' Local storage key for chat history persistence
persistHistory Optional boolean true Whether to persist chat history in local storage

Bot Icon Configuration

You can use any image format for bot icons, including animated GIFs. The widget supports various image formats and will automatically resize them to fit the chat interface.

new ChatbotWidget({
    botIcon: 'https://example.com/animated-bot.gif',
    // ... other options
});

Logo Configuration

The widget supports two types of logos:

  • Title Logo: Displayed in the chat header next to the title
  • Chat Message Logo: Optionally displayed next to bot messages in the conversation
new ChatbotWidget({
    titleLogo: 'https://example.com/company-logo.png',
    showLogoInChat: true, // Show logo next to bot messages
    tooltipText: 'Chat with our support team!',
    // ... other options
});
Tip: For best results, use square images (1:1 aspect ratio) for bot icons and horizontal/rectangular logos for title logos. The widget will automatically resize them to fit.

API Methods

The chatbot widget provides several methods for programmatic control and interaction.

Widget Instance Methods

const chatbot = new ChatbotWidget({ /* config */ });

// Get current chat ID (unique session identifier)
const chatId = chatbot.getCurrentChatId();

// Get chat history
const history = chatbot.getChatHistory();

// Update configuration at runtime
chatbot.updateConfig({
    title: 'New Title',
    primaryColor: '#ff6b6b'
});

// Clear chat history and generate new chat ID
chatbot.clearChatHistory();

// Add a message programmatically
chatbot.addMessage('Hello from the API!', 'bot');

// Show/hide suggested questions
chatbot.showSuggestedQuestions();
chatbot.hideSuggestedQuestions();

// Toggle fullscreen mode
chatbot.toggleFullscreen();

// Open/close chat (for non-inline mode)
chatbot.openChat();
chatbot.closeChat();

// Destroy the widget and clean up
chatbot.destroy();

Webhook Payload Structure

The widget sends the following payload to your n8n webhook:

{
    "message": "User's message text",
    "chatId": "unique-session-id"
}

Expected Webhook Response

Your n8n webhook should return a JSON response with either an 'output' or 'message' field:

{
    "output": "Bot response message"
}
// or
{
    "message": "Bot response message"
}

Advanced Features

Message Formatting

The widget automatically handles special characters and formatting in bot responses:

  • Line breaks: Both \n and actual newlines are converted to HTML line breaks
  • Tabs: \t characters are converted to 4 spaces
  • Escaped characters: \", \', \\ are properly unescaped
  • Carriage returns: \r characters are removed

Session Tracking

Each chat session gets a unique ID that persists across page reloads:

const chatbot = new ChatbotWidget({ /* config */ });
const sessionId = chatbot.getCurrentChatId();

// The session ID is automatically included in webhook payloads
// and stored in localStorage for persistence

Keyboard Shortcuts

  • Enter: Send message
  • Shift + Enter: New line in input
  • Escape: Exit fullscreen mode (desktop only)

Mobile Device Behavior

The widget automatically detects mobile and tablet devices and adapts its behavior:

  • Automatic Fullscreen: Mobile devices automatically enter fullscreen mode when the chat opens
  • Virtual Keyboard Handling: Input area stays visible when the virtual keyboard appears
  • Touch-Optimized: Larger touch targets and optimized scrolling behavior
  • Viewport Adaptation: Dynamically adjusts to viewport changes (e.g., browser URL bar hiding)
  • Fullscreen Restrictions: Fullscreen toggle button is hidden on mobile devices
  • Enhanced Scrolling: Ensures latest messages remain visible during keyboard interactions
Mobile Detection: The widget detects mobile devices using screen width, user agent strings, and touch support detection. Devices with screen width ‚â§ 480px are considered mobile, while 481px-1024px with touch support are considered tablets.

Auto-resize Input

The input field automatically resizes as users type, with a maximum height to prevent excessive growth.

Retry Logic

Failed requests are automatically retried with exponential backoff. Users see retry progress messages during the process.

Local Storage Structure

Chat data is stored in localStorage with the following structure:

{
    "chatId": "unique-session-id",
    "conversation": [
        {
            "text": "Message content",
            "type": "user|bot|chatbot-error",
            "timestamp": "2024-01-01T12:00:00.000Z"
        }
    ]
}

Troubleshooting

Common Issues

Webhook URL Issues:
  • Ensure your n8n webhook is active and accessible
  • Check CORS settings if hosting on different domains
  • Verify the webhook accepts POST requests with JSON payload
  • Confirm the webhook returns JSON with 'output' or 'message' field
Expected Webhook Response: Your n8n webhook should return a JSON response with either an 'output' or 'message' field: 
{
    "output": "Thank you for your message!"
}
// or
{
    "message": "Thank you for your message!"
}

Debug Mode

Enable browser console logging to debug issues:

// Check browser console for error messages
console.log('Chatbot widget loaded');

// Test webhook connectivity
fetch('your-webhook-url', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ 
        message: 'test',
        chatId: 'debug-session'
    })
})
.then(response => response.json())
.then(data => console.log('Webhook response:', data))
.catch(error => console.error('Webhook error:', error));

Performance Considerations

  • The widget automatically retries failed requests up to the configured limit
  • Chat history is stored in localStorage and has browser storage limits
  • Large bot icons may impact loading performance
  • Consider disabling history persistence for high-traffic scenarios
  • Message formatting is optimized but very long messages may impact performance
  • Mobile devices automatically enter fullscreen mode which may affect page layout
  • Virtual keyboard handling on mobile devices requires viewport adjustments

Mobile-Specific Considerations

  • Fullscreen Behavior: On mobile devices, the widget automatically enters fullscreen mode and prevents body scrolling
  • Virtual Keyboard: The widget handles virtual keyboard appearance/disappearance and adjusts the layout accordingly
  • Touch Events: Ensure your webhook can handle the increased traffic from touch-friendly interfaces
  • Viewport Units: The widget uses custom CSS properties for viewport height to handle mobile browser quirks
  • Input Constraints: Input field width is constrained on mobile devices to prevent layout issues

Browser Compatibility

The widget supports all modern browsers with ES6 module support:

  • Chrome 61+
  • Firefox 60+
  • Safari 10.1+
  • Edge 16+
